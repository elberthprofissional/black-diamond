## Feature 1: Sistema de Avaliação Pós-Atendimento

### Fluxo
1. Admin clica "Concluir Atendimento" → confirma no modal
2. Sistema envia email pro cliente com link pra avaliar 1-5 estrelas
3. Cliente abre `/avaliar/:bookingId` (público, sem login)
4. Toca nas estrelas → comenta opcionalmente → envia
5. Se 4-5 estrelas → mostra botão "Avaliar no Google Maps"
6. Dashboard mostra média de avaliação
7. TestimonialsSlider usa avaliações reais do banco

### Arquivos novos
- `src/pages/RatingPage.tsx` — página pública de avaliação
- `supabase/functions/send-rating-email/index.ts` — envia email com link
- SQL: tabela `reviews` + RPCs `get_average_rating()` + `get_top_reviews()`

### Arquivos modificados
- `src/App.tsx` — rota `/avaliar/:bookingId`

## Feature 2: Google Calendar Auto-Sync

### Fluxo
1. Booking criado → cria evento no Google Calendar do barbeiro
2. Booking cancelado → remove evento
3. Booking reagendado → atualiza evento
4. ID do evento salvo na tabela bookings (coluna `google_event_id`)

### Arquivos novos
- `supabase/functions/sync-google-calendar/index.ts` — CRUD no Google Calendar

### Arquivos modificados
- `estrutura_black_diamond.sql` — coluna `google_event_id` na tabela bookings
- `src/pages/AdminDashboard.tsx` — chama sync após criar/cancelar/reagendar
- `src/pages/BookingPage.tsx` — chama sync após criar
- `src/hooks/useReschedule.ts` — chama sync após reagendar

### Configuração manual necessária
1. Criar projeto no Google Cloud Console
2. Habilitar Google Calendar API
3. Criar OAuth 2.0 credentials
4. Autorizar uma vez (barbeiro clica num link)
5. Salvar refresh_token nos secrets do Supabase
6. Adicionar GOOGLE_CLIENT_ID, GOOGLE_CLIENT_SECRET, GOOGLE_REFRESH_TOKEN como secrets
