# Plano: Otimização PWA + Modal Full-Screen Mobile

## Contexto
O app é um PWA de barbearia. Celulares Android estão lentos com animações pesadas do framer-motion e imagens enormes. Além disso, o modal "Novo Cliente" abre como bottom sheet pela metade da tela no Android — queremos full-screen.

---

## Mudança 1: Modal "Novo Cliente" full-screen no mobile

**Arquivo:** `src/pages/AdminClients.tsx` (linhas ~836-875)

**Atual:** `flex items-end` + `w-full sm:w-[340px]` + `rounded-t-2xl` — abre como bottom sheet.

**Mudar para:** No mobile (`< sm`), o modal ocupa a tela toda com `h-full` e `rounded-none`. No desktop mantém o estilo bottom sheet/cartão.

- Trocar `flex items-end` por `items-stretch` no mobile, `items-center` no sm+
- Adicionar `h-full sm:h-auto` no painel interno
- Trocar `rounded-t-2xl sm:rounded-2xl` para `rounded-none sm:rounded-2xl`
- Trocar `border-t sm:border` para `border-none sm:border`
- Adicionar padding top seguro: `pt-safe` (já definido no CSS)

---

## Mudança 2: Desabilitar animações pesadas no mobile

**Abordagem:** Criar um wrapper que detecta `prefers-reduced-motion` e desliga animações framer-motion globalmente no mobile.

### 2a. Criar hook `src/hooks/useReducedMotion.ts`
- Retorna `true` se `matchMedia('(prefers-reduced-motion: reduce)')` ou se for mobile (viewport < 768px)
- Simples, sem dependências

### 2b. Envolver app com `MotionConfig` no `src/App.tsx`
- Importar `MotionConfig` do framer-motion
- Usar o hook para setar `reduceMotion="always"` quando apropriado
- Isso desliga springs, layout animations, exit animations em todos os `motion.*`

---

## Mudança 3: Otimizar Service Worker (cache-first para assets)

**Arquivo:** `public/sw.js`

Atualmente usa network-first para tudo (fetch primeiro, cache depois). Mudar para:
- **Assets estáticos** (imagens, fonts, JS chunks): **cache-first** — serve do cache se disponível
- **Navegação** (HTML): **network-first** com fallback pro cache (já faz isso)
- **API calls**: network-only (não cachear)
